## June 17, 2026

### What I did
- Identified the two true external inputs to the project:
  - `central_column_connectivity.csv` and `offset_column_connectivity.csv` — EM synapse counts from Shinomiya et al. 2019
  - RF parameters hardcoded in `read_RecF_data()` — distilled from calcium imaging (Arenz 2017, Drews 2020)
- Understood what the CSV files mean: 65x65 tables, rows = presynaptic cell, columns = postsynaptic cell, values = synapse count, sign = neurotransmitter type (negative = inhibitory, positive = excitatory)
- Understood the difference between zero and empty in the offset CSV (confirmed absent vs unmeasured)
- Understood how `create_multi_colM()` tiles intra and inter column matrices into the full 325x325 multi-column matrix
- Understood why `inter_colM` is divided by 2 (modeling assumption — offset CSV measures one directional connection, halved so total lateral drive per cell matches what was measured)

### Key concepts understood
- The paper's main argument: transient response behavior in most medulla neurons is inherited from L1/L2 via the H-current (Ih), not created by feedback loops
- All pre-built `.npy` files were generated by Borst's code — only the CSVs and RF parameters are true external inputs
- Figure 1 is entirely independent of the CSVs — it only uses the RF parameters
- `read_ConnMs()` exists but is never called in the current codebase — the `.npy` circuit files are pre-distributed, the saving step is missing from the repo

### Questions / things to follow up
- Why exactly is inter_colM divided by 2? Need to check Shinomiya et al. 2019 to confirm how inter-column synapses were counted
